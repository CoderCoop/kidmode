Try it, step by step

Everything below happens inside the phone on the left.

  1. Tap and mash the screen.

    Ripples bloom under every touch. Hammer it with many fingers — the animation pool never drops a frame.

  2. Switch activities with the chips up top.

    💧 Ripples, 🎈 Pop (bursting shapes + haptics on a real device), 🎹 Sounds (a tap-anywhere pad that actually plays tones here).

  3. Try to leave — you can’t.

    There’s no back button, no home. On a real device the OS is pinned away entirely.

  4. Open the grown-up gate.

    Press and hold “◉ hold to exit” for ~1.5s (this stands in for the two-finger long-press). Toddlers don’t hold deliberately.

  5. Tap the four corners clockwise from the top-left.

    Wrong order or dawdling resets it — random mashing can’t solve an ordered, timed sequence.

  6. Enter the PIN 2-4-6-8.

    The pad reshuffles every open and after any mistake, so it can’t be learned as a finger-path.

  7. You’re out. Tap Resume play to re-lock.

     

What you’re seeing

The four requirements, each demonstrated above.

🔒
Device lockdown

Native screen-pinning (Android) / Guided Access (iOS); notifications, gestures, and hardware buttons suppressed.

🚪
Parental gate exit

Multi-finger long-press → clockwise corners → randomized PIN. The only way out.

🌊
Resilient canvas

A fixed animated pool absorbs aggressive multi-touch with zero re-renders or GC churn.

🧩
Modular activities

Ripples, Pop, Sounds are pluggable modules from a registry — add one file to ship another.

This is an HTML re-creation for the browser. The shipping app is React Native + TypeScript — see README.md and docs/ARCHITECTURE.md.
